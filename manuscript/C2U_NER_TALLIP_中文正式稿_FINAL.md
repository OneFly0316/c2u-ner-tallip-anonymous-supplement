# C2U-NER：低资源维吾尔语命名实体识别的汉维资源构建与边界评测

## 摘要

低资源维吾尔语命名实体识别面临双重困难：一方面，目标语言缺少规模足够、来源可说明且边界可靠的标注资源；另一方面，维吾尔语黏着形态会使实体与格、数、所属和方位后缀连续结合，常规 token 级评价难以反映真实边界错误。本文构建 C2U-NER，一个由中文新闻 NER 资源经汉维跨语言转换得到的维吾尔语实体识别数据集，并提出可审计的迭代式数据蒸馏流程。该流程依次结合 Hunyuan-7B 翻译、占用掩码硬对齐、Qwen3-80B 形态感知软对齐、生成式实体回填和 XLM-R 判别式自举，将实体保留率从硬对齐阶段的 65.34% 提升到最终 96.92%。C2U-NER 包含 27,818 条维吾尔语句子和 44,118 个 PER、LOC、ORG 实体，自动 span 审计范围内坏字符区间数为 0。基于严格字符级完全匹配评价，TBRK-IBP-CINO 在 C2U-NER 上取得 77.69 的三随机种子平均 F1，较 IBP-CINO 提升 1.15 个百分点，并在三个公开来源整理的外部维吾尔语验证集上呈现一致提升。结果表明，低资源黏着语 NER 需要把数据构建、边界审计和模型评价作为整体问题处理。

**关键词**：维吾尔语；低资源语言；命名实体识别；跨语言标注投影；数据蒸馏；字符级边界评测

**Keywords**：Uyghur; low-resource language; named entity recognition; cross-lingual annotation projection; data distillation; character-level boundary evaluation

**ACM CCS Concepts**：Computing methodologies -> Natural language processing；Information systems -> Information extraction；Computing methodologies -> Machine translation；Applied computing -> Language translation。

## 1 引言

命名实体识别（Named Entity Recognition, NER）是信息抽取、知识图谱构建、跨语言检索、问答系统和检索增强生成中的基础任务。在英语、汉语等高资源语言中，公开标注语料、共享评测任务和预训练语言模型已经较为成熟，模型改进通常可以在稳定数据和统一协议下比较。但在许多亚洲低资源语言中，NER 的首要障碍并不是模型容量不足，而是缺少可训练、可评测、可复核的数据资源。如果数据规模过小、来源不清或边界标注不稳定，即使模型分数较高，也很难判断其对真实信息抽取任务是否可靠。

维吾尔语 NER 集中体现了这一问题。维吾尔语属于突厥语族，具有黏着语特征，名词和专名后可连续附加格、数、所属、人称及方位后缀。实体在句子中往往不以孤立词形出现，而是与功能后缀、属格结构、行政层级词或机构职能词组合为更长的表面形式。例如，地名可与方位后缀粘连，机构名可被所属结构扩展为长名词短语。通用子词分词能够缓解未登录词问题，但并不能保证模型输出与真实字符边界一致。对于下游实体链接、检索索引和知识库写入而言，少识别一个后缀或多吸收一个修饰词都会造成可见错误。

跨语言迁移为低资源 NER 提供了可行路径。已有研究通常采用两类策略：一类利用 mBERT、XLM-R、CINO 等多语言或区域语言预训练模型进行表示迁移；另一类通过机器翻译、词对齐、平行语料或 Wikipedia 跨语言链接将源语言标注投影到目标语言。后一类方法能够显著降低人工标注成本，但其有效性依赖一个强假设：源端实体在目标端译文中能够被稳定定位。汉语到维吾尔语的转换恰恰会系统性破坏这一假设。汉语实体常以相对独立的词或短语出现，而维吾尔语译文中实体可能发生语序重排、后缀粘连、转写变体、简称扩展、意译或漏译。直接采用字符串匹配或一次性词对齐，往往会得到高精度但低召回的投影结果。

因此，面向维吾尔语 NER 的数据构建不能被简化为“翻译中文句子并复制标签”。一个能够支撑严谨实验的数据集至少需要满足四项条件：源端实体在目标端尽可能保留；目标端实体具有可程序核验的字符边界；构建过程能够定位并修复漏译、错译和边界不完整样本；最终数据支持严格评价，而不只是 token 级近似匹配。本文围绕这些要求构建 C2U-NER，并将其作为低资源黏着语实体识别中的资源构建与边界评测问题来研究。

本文采用迭代式数据蒸馏（Iterative Data Distillation, IDD）构建 C2U-NER。IDD 不把大语言模型的翻译输出直接视为最终标注，而是将其作为候选中间产物，再经过硬对齐、软对齐、生成式修复和判别式自举，逐步转化为可审计的目标端标注。具体而言，Hunyuan-7B 负责生成维吾尔语候选译文；占用掩码硬对齐用于高精度锚定字面稳定实体；Qwen3-80B 形态感知软对齐用于恢复后缀粘连和转写变体；生成式回填用于修复漏译实体；XLM-R 判别式模型则利用高置信样本补足剩余长尾对齐。每一阶段均通过连续子串检查、占用冲突检查和坏 span 审计约束输出质量。

除数据构建外，本文还将评测协议作为核心问题处理。维吾尔语实体识别中的许多错误并非类型判断错误，而是边界偏移。若仅采用 token 级 BIO 评价，模型可能因识别到实体附近区域而获得较高分数，但输出片段仍不能直接用于下游系统。为此，本文采用严格字符级完全匹配：预测实体只有在起点、终点和类型均与参考标注一致时才计为正确。为了评估 C2U-NER 暴露出的边界问题是否具有模型研究价值，本文进一步构建 IBP-CINO、TBR-IBP-CINO 和 TBRK-IBP-CINO 三阶段模型链，分别处理 token 到字符 span 的恢复、局部边界修正和候选质量过滤。

围绕上述问题，本文设置四个研究问题。**RQ1：** 在汉语到维吾尔语的跨语言转换中，多阶段数据蒸馏能否在保持实体语义的同时生成合法字符级 span？**RQ2：** 与已有公开维吾尔语 NER 资源相比，C2U-NER 在样本规模、实体规模和边界审计方面提供了怎样的增量？**RQ3：** 严格字符级评价下，边界校准模型是否能稳定改善 C2U-NER 上的实体抽取结果？**RQ4：** C2U-NER 上观察到的边界校准趋势能否迁移到不同来源的公开维吾尔语 NER 数据？这四个问题分别对应本文的数据构建、资源比较、主实验和外部验证。

本文贡献可以概括为四点。第一，构建 C2U-NER 数据集，提供 27,818 条维吾尔语句子和 44,118 个严格字符级实体标注，并在本文统一协议下相较现有公开维吾尔语 NER 资源提供更大的样本规模和实体规模。第二，提出可审计的汉维实体数据蒸馏流程，将大语言模型的生成能力与规则校验、判别式自举结合，使最终实体保留率达到 96.92%。第三，建立适合维吾尔语黏着形态的严格字符级 NER 评测协议，并给出实体长度、空样本和坏 span 等数据画像。第四，通过 C2U-NER 和三个公开来源外部验证集表明，候选保留门控能够在严格字符级评价中减少误报并提升 F1。

本文关注的核心问题是低资源语言资源构建与评价，而不是单纯追求新的模型结构。C2U-NER 的价值在于为维吾尔语 NER 提供可复核的数据构建路径、可审计的字符级标注和可复现的边界评测协议；IBP/TBR/TBRK 实验用于验证该资源暴露的边界问题会实际影响模型行为，并为后续低资源黏着语 NER 研究提供基线和错误诊断。

## 2 相关工作

### 2.1 低资源与跨语言命名实体识别

NER 的经典研究从共享任务和序列标注模型出发，逐步形成以实体类型、边界识别和标准化评价为核心的任务范式。CoNLL-2003 推动了语言无关 NER 评测 [1]，BiLSTM-CRF 等神经序列标注模型进一步减少了手工特征依赖 [2]。上下文表示随后显著改变了 NER 的建模方式：ELMo 和 contextual string embeddings 证明了深层上下文特征对序列标注的价值 [16,17]，BERT 及其后续分析表明 Transformer 预训练能够提供强上下文表示 [3,18]。在跨语言场景中，mBERT、XLM 和 XLM-R 展示了多语言预训练的迁移能力 [4,19-22]，CINO 则面向中文少数民族语言构建预训练表示，更接近维吾尔语、藏语、蒙古语等区域语言场景 [5]；Glot500 等覆盖更多低资源语言的模型进一步扩展了多语表示的语言范围 [15]。这些模型为低资源 NER 提供了基础表示，但仍依赖目标语言数据和明确评价协议。

跨语言 NER 的另一条路线是数据迁移。代表性工作通过翻译、词对齐、Wikipedia 链接或多语言知识库将源语言实体标注投影到目标语言 [6,7,23,24]。WikiANN/PAN-X 将跨语言名称标注扩展到大量语言，XTREME 也将其纳入跨语言泛化评测 [6,9]。这类资源覆盖范围广，但落到具体低资源语言时，仍可能受到样本规模、领域分布和边界粒度的限制。低资源 NLP 综述也指出，数据可获得性、标注规范和语言覆盖不均衡仍是低资源研究的核心限制 [25,26]。C2U-NER 与这些资源互补：本文不追求覆盖更多语言，而是围绕汉维转换和维吾尔语边界问题构建更大规模、更便于边界诊断的目标语言资源。

### 2.2 翻译投影与自动数据构建

翻译投影的基本思想是利用源语言标注和跨语言对齐关系生成目标语言训练数据。该方法成本较低，适合低资源场景，但容易受到机器翻译质量、语言结构差异和对齐错误影响。早期弱监督跨语言 NER 研究表明，投影质量不仅取决于翻译或词对齐本身，也取决于目标端表示、噪声过滤和实体边界恢复策略 [23,24]。对于汉维语言对，语序变化、形态粘连和专名转写会使源端实体难以被直接映射到目标端字符区间。已有汉维 NER 数据增强研究表明，机器翻译可以改善维吾尔语实体识别训练数据不足的问题，但翻译生成标签仍需要进一步处理边界偏移和漏译 [8]。

本文的 IDD 流水线将传统“翻译后投影”改造为多阶段数据蒸馏。硬对齐阶段强调高精度，软对齐阶段强调形态和语义对应，生成式修复阶段强调实体保留，判别式自举阶段强调长尾补召回。这样，数据构建不再是一次性操作，而成为可记录、可审计、可诊断的过程。同时，每个阶段的作用都可以通过实体召回率、坏 span 数和冲突检查结果量化，从而降低自动构造数据的不透明性。

### 2.3 维吾尔语和黏着语实体边界

维吾尔语的黏着语形态使实体边界成为 NER 的关键问题。已有维吾尔语 NER 研究关注子词表示对形态稀疏性的缓解作用 [10]，但子词表示并不等同于字符边界恢复。跨语言表示研究也反复指出，多语言模型的零样本迁移能力并不均匀，目标语言的书写系统、预训练覆盖和分词方式会影响下游表现 [20-22]。对于地名、机构名和长实体，模型可能已经识别到实体中心词，却未能判断后缀或修饰成分是否属于实体 span。尤其在严格信息抽取场景中，边界偏移会直接影响实体链接和知识库写入。

因此，字符级完全匹配被用作本文的主评价标准。该标准要求预测实体的起点、终点和类型同时正确，比 token 级 BIO 评价更严格，也更符合下游系统对实体片段的需求。C2U-NER 中大量中长实体和接近 40% 的空样本比例，使其不仅能评价实体召回，还能评价模型对长边界和无实体句子的误报控制能力。

### 2.4 大语言模型辅助低资源数据构建

大语言模型具备较强的多语言翻译、指令遵循和上下文推理能力，近年来开始被用于低资源数据增强和弱监督标注。Hunyuan-7B-Instruct 和 Qwen3 等公开模型或技术报告为本文的离线翻译与对齐组件提供了可追溯来源 [11,12]。然而，NER 数据构建不同于普通文本生成：它要求句子自然、实体语义保留、类型一致、边界可定位且多个实体之间不发生冲突。LLM 生成结果若缺少约束，可能出现漏译、幻觉、边界不可定位或实体替换。

在本文流程中，LLM 是离线数据构建组件，而不是最终在线抽取器。Hunyuan-7B 用于生成候选译文，Qwen3-80B 用于形态感知对齐和缺失实体修复；进入数据集的输出均需通过程序化审计。这一设计利用了 LLM 处理复杂语义转换的能力，同时避免把不可控生成结果直接作为训练标签。与此同时，数据资源建设还需要记录数据来源、标注假设、潜在偏差和适用边界。Data Statements、Datasheets for Datasets 和 Model Cards 等工作为 NLP 数据集和模型披露提供了通用框架 [27-29]，本文的数据卡、来源审计和 AI 工具使用声明也遵循这一思路。

## 3 C2U-NER 数据集构建

### 3.1 任务定义

给定中文句子 \(X_{zh}\) 及其实体集合 \(Y_{zh}=\{(s,e,c)\}\)，其中 \(s,e\) 表示源端字符边界，\(c\in\{PER, LOC, ORG\}\)，C2U-NER 的构建目标是生成维吾尔语句子 \(X_{ug}\) 及目标端实体集合 \(Y_{ug}\)。目标实体需要满足三项约束：实体文本是 \(X_{ug}\) 的连续子串；字符区间不越界、不为空且不与其他实体非法冲突；实体类型与源端语义一致。

本文从三个指标约束数据质量。实体保留率衡量源端实体被成功转换为目标端合法 span 的比例；span 合法性衡量目标端边界是否可由程序稳定读取；边界可评测性衡量数据是否适合严格字符级完全匹配，而非只能进行 token 级近似评价。

### 3.2 迭代式数据蒸馏流程

![](figures/fig1_idd_pipeline_final.png){width=6.2in}

**图1 | C2U-NER 的迭代式数据蒸馏流程。** 流水线从中文 NER 标注出发，经零样本翻译、掩码硬对齐、形态感知软对齐、生成式修复回填和判别式自举获得维吾尔语实体 span。图中展示了每个阶段的输入、输出和质量控制信号，突出 C2U-NER 不是一次性翻译投影数据，而是经过多阶段实体追踪、修复和审计后形成的可复核资源。

第一阶段是零样本翻译。系统使用 Tencent Hunyuan-7B-Instruct 将中文句子翻译为维吾尔语候选句。该阶段只生成语义自然的目标端文本，不直接产生最终标注。为降低生成随机性，构建过程采用固定指令模板和贪婪解码，并清理非必要特殊 token。该阶段可能产生漏译、转写差异或实体形态变化，因此后续阶段需要继续追踪源端实体。

第二阶段是占用掩码硬对齐。对每个源端实体，系统先获得其维吾尔语字面译名，再在译文中搜索候选片段。为避免多个实体复用同一字符区域，系统维护布尔占用掩码 \(\mathbf{M}\)。只有当候选区间未被占用且文本连续出现在译文中时，才接受该匹配。硬对齐精度高、可解释性强，但难以处理后缀粘连、长距离重排和转写变体。构建记录显示，仅依靠硬对齐时实体召回率约为 65.34%。

第三阶段是形态感知软对齐。对于硬对齐失败的实体，系统调用 Qwen3-80B，并在提示中提供中文句子、源端实体、实体类型、维吾尔语候选句和已占用 span。模型需要返回目标句中与源端实体语义对应的完整片段，而不只是词根。软对齐结果仍需通过连续子串检查和占用冲突检查。该阶段主要处理维吾尔语实体词干与后缀粘连、转写变体和局部重组。

第四阶段是生成式修复回填。对于仍包含缺失实体的样本，系统要求大语言模型在保持译文自然性的前提下恢复缺失实体，并输出修正后的维吾尔语句子和实体映射。修复结果并不直接进入数据集，而是重新经过 span 解析、占用检查和类型一致性校验。该阶段对召回率提升最为关键，使实体对齐召回率从 65.34% 提升到 94.87%。

第五阶段是判别式自举。系统使用前四阶段得到的高置信样本训练轻量 XLM-R 判别式模型，对剩余低置信或未对齐样本进行残差推理。只有当预测置信度超过阈值且通过合法性审计时，实体才会被合并回数据集。该阶段最终将实体保留率提升至 96.92%。通过“LLM 生成 + SLM 清洗”的组合，自动数据构建同时获得语义修复能力和程序化质量控制。

### 3.7 抽样人工复核协议

由于 C2U-NER 的目标端数据由自动流水线构建，抽样人工复核用于补充程序化审计无法覆盖的语义质量证据。人工复核被定位为发布材料的一部分，并与已完成的自动审计区分开来：自动审计证明 span 合法、标签合法和切片一致，人工复核则进一步判断翻译语义、实体类型和边界语义是否合理。复核对象覆盖测试集和少量训练集样本，优先抽取四类高风险样本：包含长实体的样本、IDD 修复阶段回填实体的样本、LOC/ORG 边界较长的样本，以及空样本或模型误报较多的样本。每个样本的复核项包括：维吾尔语句子是否保留源端核心语义，实体文本是否为目标句连续子串，实体边界是否包含必要后缀或修饰成分，实体类型是否与源端一致，以及是否存在明显漏译、错译或不自然表达。

## 4 字符级评测与边界校准模型

### 4.1 严格字符级评价

本文采用严格字符级实体完全匹配作为主评价标准。每个预测实体表示为 \((s,e,c)\)，其中 \(s\) 和 \(e\) 为字符起止位置，\(c\) 为实体类型。只有当预测实体的起点、终点和类型均与参考标注一致时，才计为 true positive。若实体类型正确但边界少一个后缀，或边界正确但类型错误，均计为错误。该标准比 token 级 BIO 评价更严格，但更符合实体链接和知识库写入对片段准确性的需求。

### 4.2 边界校准模型链

![](figures/fig3_model_chain_final.png){width=6.2in}

**图3 | 错误驱动的边界校准模型链。** CINO 编码器生成上下文 token 表示，IBP-CINO 从 token 候选恢复字符级 span，TBR-IBP-CINO 对局部左右边界进行扩展或收缩，TBRK-IBP-CINO 通过 keep gate 过滤低质量候选，最终在严格字符级完全匹配协议下评价实体边界和类型。模型链与错误类型一一对应：token-to-span mismatch 对应 IBP，boundary shift 对应 TBR，spurious candidate 和 empty-sample false positive 对应 TBRK。

需要说明的是，IBP/TBR/TBRK 在本文中不是作为主要算法贡献提出，而是作为边界感知诊断基线链使用。其作用是检验 C2U-NER 是否能够暴露严格字符级 NER 中的 token-to-span mismatch、boundary shift 和 spurious candidate 等错误，并为后续低资源黏着语 NER 研究提供可复现的比较起点。

本文选择 CINO-base-v2 作为后续边界校准模型的统一编码器，主要基于两点考虑。第一，CINO 面向中文少数民族语言进行预训练，覆盖维吾尔语等区域语言，较通用多语言模型更贴近本文目标语言场景 [5]。第二，为避免骨干差异干扰边界机制分析，本文在同一数据划分、同一 IBP 边界恢复机制和同一严格字符级评价协议下比较候选编码器。结果显示，CINO 在验证集选择标准上最稳定，测试集严格字符级 F1 也略高于 Glot500 和 XLM-R [4,15]。因此，后续 IBP、TBR 和 TBRK 均固定 CINO 作为编码器；该选择用于控制变量，而不意味着本文主张 CINO 在所有维吾尔语 NER 场景中均显著优于其他 backbone。

IBP-CINO 以 CINO-base-v2 为编码器，在 token 级候选基础上结合 token offset 和边界指针输出字符级实体 span。该模块主要解决评价接口问题：模型输出需要从 BIO 标签转化为可审计的字符区间。

TBR-IBP-CINO 在 IBP 基础上加入 token boundary refiner。给定候选实体，TBR 在左右边界附近的局部窗口中判断是否需要向外扩展、向内收缩或保持不变。该模块对应 C2U-NER 中常见的边界欠扩展和过扩展错误。

TBRK-IBP-CINO 进一步加入 token refine keep gate，对修正后的候选实体进行保留或拒绝判断。该模块针对的是 spurious entity、空样本 false positive 和边界修正后产生的低质量候选。三阶段模型链按“恢复字符 span、校准局部边界、过滤低质量候选”的顺序工作。

## 5 实验设置

主实验使用 C2U-NER 的 train、dev 和 test 划分。外部验证使用三个公开来源整理得到的维吾尔语数据集：`wikiann_ug_per_org_loc` 来自 WikiANN/PAN-X 维吾尔语数据；`finer_ug_per_org_loc` 基于 Hugging Face 数据集 `whoisjones/fiNERweb` 的 `uig` 数据处理得到；`codemurt_uyghur_ner_per_org_loc` 基于 Hugging Face 数据集 `codemurt/uyghur_ner_dataset` 整理得到。三者均映射到 PER、ORG、LOC 标签空间，并转换为本项目统一的字符 span 格式。外部验证集不用于提出公开榜单排名，只用于检验 C2U-NER 上观察到的边界校准趋势是否能在不同来源的维吾尔语数据上复现。

模型训练采用随机种子 42、2024 和 3407。最大输入长度为 512。TBR 的局部边界修正窗口设为 2，默认 refine 阈值为 0.55；TBRK 默认 keep 阈值为 0.50。dev-selected 结果通过验证集阈值搜索选择 refine 和 keep 阈值，再在测试集报告。所有结果均使用相同严格字符级评价脚本计算 Precision、Recall 和 F1。

为评估结果稳定性，本文对 seed2024 结果进行成对 bootstrap 检验。C2U-NER bootstrap 使用 10,000 次采样。错误分析基于实体级 TP、FP 和 FN，并进一步拆分 spurious entity、empty-sample false positive、边界偏移、类型混淆、missed entity 和长实体漏检等类别。

实验复现按三个层次组织。第一，数据转换层复现公开来源数据到统一字符 span 格式的过程，包括标签映射、句子重建和坏 span 审计。第二，模型训练层复现 C2U-NER 上 IBP、TBR 和 TBRK 三随机种子训练及验证集阈值选择。第三，统计分析层复现主结果表、外部验证表、bootstrap 置信区间和错误类型分解。本文配套材料整理对应脚本、随机种子、模型 checkpoint 名称、主要超参数和输出文件校验说明，以便读者区分数据构建、模型训练和统计检验三个可复现环节。

实验结果围绕研究问题展开。第 6.1 节回答 RQ1，分析 IDD 各阶段对实体保留和坏 span 审计的作用；第 3.5-3.7 节和表4-表6 回答 RQ2，比较 C2U-NER 与公开维吾尔语 NER 资源的规模、用途、类型组成、资源发布口径和人工复核协议；第 6.2-6.6 节回答 RQ3，报告骨干选择、主实验、过滤基线、消融和错误分析；第 6.4 节回答 RQ4，检验外部公开来源上的一致性。

## 6 结果与分析

### 6.1 C2U-NER 的资源价值

C2U-NER 的阶段性构建结果回答了 RQ1。汉维实体投影的主要难点在于实体保留和目标端边界恢复。硬对齐阶段召回率仅为 65.34%，说明字符串级映射难以覆盖维吾尔语中的形态变化和翻译重组。生成式回填将召回率提升到 94.87%，判别式自举进一步提升到 96.92%，说明多阶段蒸馏流程能够减少漏译和长尾对齐失败。在本研究审计范围内三个划分坏 span 数均为 0，支持后续严格字符级评测。

**表7 | IDD 各阶段对实体保留率的贡献。**

| 构建阶段 | 主要作用 | 实体保留率 | 阶段收益 |
|---|---|---:|---:|
| 占用掩码硬对齐 | 高精度定位字面稳定实体 | 65.34% | 基础锚定 |

### 6.2 骨干编码器选择与边界恢复必要性

在进入 TBR 和 TBRK 之前，骨干编码器选择实验用于回答为什么后续基线固定使用 CINO。直接采用普通 token-classification 模型进行 BIO 解码时，不同 tokenizer 的 offset 行为会显著影响字符级 span 还原：CINO 在 dev token-F1 上最高（90.58），但普通 BIO 到字符 span 的直接转换无法稳定恢复 token 内边界；XLM-R 在这一诊断设置下 test char-F1 更高（69.12）。这一现象说明，严格字符级评价不能只依赖常规 token 级序列标注结果，也支持本文引入 IBP 作为统一的 token-to-span 边界恢复接口。

为了排除解码接口差异，本文在同一 IBP 机制下比较 CINO-base-v2、XLM-R-base 和 Glot500-base。表8 显示，CINO 的 best dev char-F1 为 77.25，高于 Glot500 的 76.19 和 XLM-R 的 74.76；在测试集上，CINO 的严格字符级 F1 为 76.08，略高于 Glot500 的 76.00，并高于 XLM-R 的 73.30。由于模型选择应依据验证集表现完成，且 CINO 的预训练语料面向中文少数民族语言场景，本文后续固定 CINO 作为 IBP/TBR/TBRK 的编码器。需要强调的是，Glot500 在测试集上与 CINO 非常接近。因此，该实验只能支持“CINO 在本文 dev-selected IBP 协议下更稳且更贴近任务场景”，不能被解释为 CINO 在所有骨干模型上都具有统计优势。

### 6.5 过滤基线与消融分析

为回应 TBRK 是否只是普通阈值过滤的问题，本文在 seed2024 上补充过滤基线和阈值消融。比较对象包括三类：第一，IBP-CINO 不使用局部边界修正和候选过滤；第二，TBR-IBP-CINO 只使用局部边界修正，并通过验证集选择 refine 阈值；第三，TBRK-IBP-CINO 在 TBR 候选上增加候选级 keep gate，并进一步在验证集上选择 refine/keep 阈值。这里的目的不是把 TBRK 包装成主要模型贡献，而是检验候选质量建模是否比普通阈值调节提供额外诊断价值。

结果见表12。IBP-CINO 在 seed2024 上的严格字符级 F1 为 76.89；TBR-IBP-CINO 通过 refine 阈值选择后提升到 77.30，说明仅靠边界局部修正可以带来有限收益。TBRK 默认 keep=0.50 时达到 77.65，已经高于 TBR；在验证集上联合选择 refine=0.50、keep=0.80 后，测试集 F1 达到 77.94。验证集阈值扫描显示，keep 阈值升高通常提高 precision、降低 recall，且 F1 并非单调上升；例如 validation 上 keep=0.90 时 precision 最高但 F1 低于 keep=0.80。这说明 TBRK 的收益不能简单解释为“把低置信预测全部删掉”，而是来自 TBR 候选生成、候选级保留判断和验证集选择三者的组合。

| 设置 | refine 阈值 | keep 阈值 | Precision | Recall | F1 | TP/FP/FN | 解释 |
|---|---:|---:|---:|---:|---:|---:|---|
| IBP-CINO | - | - | 76.58 | 77.19 | 76.89 | 5642/1725/1667 | 无边界修正和候选过滤 |
| TBR-IBP-CINO | 0.50 | - | 76.94 | 77.66 | 77.30 | 5676/1701/1633 | 仅局部边界修正 |

同时，早期通用增强模块的消融结果提供了一个反向证据：在翻译投影得到的维吾尔语 NER 数据上，简单叠加 MSD、DMA、FGM 等通用模块并不稳定优于 plain transformer baseline。CINO greedy 的单次 F1 为 81.34，而 CINO full enhanced 为 79.77；XLM-R family 中也出现 full enhanced 低于某些删减配置的现象。本文不把这组旧实验作为主结果使用，但把它作为方法定位的依据：当前主要瓶颈更接近数据噪声、字符边界和候选质量，而不是堆叠更多通用神经模块。因此，后续主实验采用 IBP/TBR/TBRK 这条错误驱动的诊断基线链。

### 6.6 错误分析

C2U-NER seed2024 的错误分析表明，TBRK 的主要收益来自 false positive 减少。IBP-CINO 的 FP 为 1,725，TBR-IBP-CINO 降至 1,701，TBRK-IBP-CINO 进一步降至 1,550，较 IBP 减少 175 个。与此同时，FN 从 1,667 降至 1,652，仅减少 15 个（图5）。这说明 TBRK 的收益并非来自无条件扩大召回，而主要来自 keep gate 对候选实体的质量校准。

![](figures/fig5_error_counts.png){width=5.8in}

**图5 | TBRK 的主要收益来自候选误报抑制。** a，IBP-CINO 与 TBRK-IBP-CINO 在 C2U-NER seed2024 测试集上的 TP、FP、FN 对比。b，TBRK 在 spurious entity、empty-sample false positive 和 both-boundary shift 三类误报上均减少。c，剩余错误主要表现为 missed entity 与长实体漏检增加，提示 keep gate 提升精确性的同时仍需进一步改善复杂边界召回。

按实体类型拆分后，TBRK 对 LOC 和 ORG 的收益大于 PER。PER F1 从 87.61 提升到 87.96，LOC 从 75.84 提升到 76.96，ORG 从 69.26 提升到 70.69。LOC 和 ORG 更常出现方位后缀、属格结构、多词修饰和机构词扩展，因此更依赖边界校准。剩余错误主要集中在 missed entity 和长实体召回，说明下一步需要在保持误报抑制能力的同时增强高召回 span 候选生成。

## 7 讨论

C2U-NER 的主要意义在于将维吾尔语 NER 的数据瓶颈具体化为可审计的数据构建问题。与只报告模型结果不同，本文说明目标语言训练数据如何获得、源端实体如何保留、目标端边界如何核验，以及所得数据如何支撑严格评价。与现有公开维吾尔语 NER 资源相比，C2U-NER 同时提供更大的样本规模、实体规模和面向汉维转换的边界审计过程。实体保留率、坏 span 审计、实体长度分布和空样本比例共同说明，C2U-NER 覆盖了低资源黏着语实体识别中的关键失败场景。

从方法角度看，IDD 流水线展示了一种适合低资源语言资源构建的折中方案。大语言模型用于处理翻译、重组和漏译修复，规则审计用于约束边界合法性，判别式模型用于补足长尾实体。这种组合比单次翻译投影更稳健，也比完全人工标注成本更低。其局限在于，自动构建数据仍可能保留翻译偏差和风格偏差，因此 C2U-NER 的资源定位是“自动构建并经程序化审计的派生数据资源”，而不是完全人工标注语料。数据卡、许可证说明、人工复核比例和一致性统计构成资源发布材料的一部分，并与模型代码和复现实验脚本共同提供。

模型实验说明，C2U-NER 暴露出的边界问题具有外部分析价值。TBRK-IBP-CINO 不仅在 C2U-NER 上优于 IBP-CINO，也在三个公开来源整理得到的外部维吾尔语验证集上保持同向提升。与此同时，错误分析显示 TBRK 的主要收益来自误报抑制，而不是长实体召回。这一结果提示，低资源黏着语 NER 后续需要同时改进高召回候选生成和候选校准机制。

本文仍有若干限制。第一，C2U-NER 当前只覆盖 PER、LOC 和 ORG，尚未扩展到时间、事件、产品和民族文化专名等细粒度类别。第二，维吾尔语侧主要由自动流程构建，虽然进行了程序化审计并设计了抽样人工复核协议，但当前复核清单尚未填入双人判断结果，因而本文不能报告实际复核通过率、一致性统计和错误类型比例。第三，外部验证数据经过统一标签映射和字符 span 转换，因此结果应解释为统一协议下的外部一致性验证，而不是与原始数据集榜单直接比较。第四，TBRK 对 missed entity 和长实体漏检改善有限，后续需要进一步研究长度感知 span 生成和候选重排序。这些限制不否定 C2U-NER 作为资源构建研究的价值，但要求本文结论限定在 PER/LOC/ORG 实体、新闻源端语料和严格字符级 NER 场景内。

## 8 有效性威胁与伦理考虑

### 8.1 内部有效性

本文的内部有效性主要受到自动数据构建误差影响。虽然 IDD 流水线通过占用掩码、连续子串检查、span 越界检查、标签空间检查和判别式自举减少了明显错误，但目标端维吾尔语句子仍可能保留机器翻译偏差、表达风格偏差或实体语义细微偏移。为降低这一风险，本文不把大语言模型输出直接视为最终标注，而是将其置于多阶段审计流程中；同时，实验报告实体保留率、坏 span 数、实体长度分布和空样本比例，使读者可以判断数据构建质量。公开版本将提供抽样人工复核记录，用于进一步量化语义等价性和边界一致性。

另一个内部有效性风险来自评测脚本和阈值选择。本文所有模型均使用同一严格字符级完全匹配脚本，并在相同 train/dev/test 划分上比较；TBRK 的 dev-selected 结果仅使用验证集选择阈值，再在测试集报告。seed2024 的 bootstrap 检验和三随机种子平均结果用于降低单次运行偶然性。尽管如此，阈值搜索范围、候选生成策略和随机种子数量仍会影响最终数值，因此论文公开材料提供完整阈值搜索配置、随机种子和中间预测文件。

### 8.2 外部有效性

C2U-NER 的源端来自本地整理的人民日报中文 NER 标注资源，目标端由汉维转换生成，因此其领域分布和语言风格可能更接近新闻文本，而不一定覆盖社交媒体、口语文本、历史文献或专业领域文本。外部验证集覆盖 WikiANN/PAN-X、fiNERweb-uig 和 codemurt 三个公开来源，能够说明边界校准趋势并非只存在于 C2U-NER 单一分布；但这些数据集本身规模、标注策略和领域仍有限。因此，本文只表述为“在新闻派生资源和三个公开来源验证集上观察到同向趋势”，不对所有维吾尔语 NER 场景作无条件泛化。

此外，C2U-NER 只覆盖 PER、LOC 和 ORG 三类实体。该标签空间与传统 NER 评测兼容，便于和公开数据统一比较，但无法覆盖民族文化专名、宗教机构、事件、作品、产品、时间和数值表达等更细粒度类别。对于实际应用而言，未来需要在 C2U-NER 的构建流程基础上扩展类型体系，并重新评估实体保留、边界合法性和类型一致性。

### 8.3 数据伦理与资源治理

维吾尔语属于低资源少数民族语言，相关数据资源建设应避免将语言社群仅作为技术基准对象。低资源 NLP 和语言多样性研究已经指出，语言覆盖不足会影响技术公平性，但面向弱势语言的资源建设也必须同时处理数据来源、代表性和潜在误用风险 [25,26]。本文构建数据的目的，是改善低资源语言信息处理条件，而不是进行个人身份推断、敏感属性识别或群体画像。C2U-NER 仅包含 PER、LOC 和 ORG 三类传统命名实体，且源端来自新闻领域文本；发布时遵守源端数据许可、模型使用条款和目标平台数据政策。若数据中包含可识别个人姓名，公开版本将在数据卡中说明来源、用途、潜在风险和使用限制 [27,28]。

大语言模型辅助构建也可能引入翻译偏差和文化表达偏差。对于低资源语言资源，单纯追求规模可能放大不自然译文或不符合目标语言习惯的表达。本文通过形态感知软对齐、生成式修复审计和坏 span 检查控制边界质量，但这些程序化检查不能完全替代母语者质量评估。因此，C2U-NER 的发布材料包含数据卡、构建流程说明、抽样复核协议和已知偏差说明，并鼓励后续研究者在具体应用前进行领域和语言质量复核。

### 8.4 可复现性威胁

本文使用的大语言模型、预训练编码器和公开数据集均可能随时间更新，导致完全复现实验时出现版本差异。为降低这一风险，本文固定模型名称、checkpoint 版本、数据下载时间、随机种子、阈值搜索范围和评价脚本版本，并在模型披露上参考 Model Cards 所强调的模型来源、用途和限制记录 [29]。对于无法长期稳定复现的在线资源，匿名审稿或正式发布材料提供可校验的处理后数据摘要、哈希值或归档文件。本文报告的关键中间数值包括硬对齐召回率 65.34%、生成式修复后召回率 94.87%、最终实体保留率 96.92%、坏 span 数 0 以及 TBRK-vs-IBP 的 bootstrap 置信区间，这些数值可作为复现实验的校验点。

## 9 结论

本文构建了 C2U-NER，一个面向低资源维吾尔语命名实体识别的汉维跨语言实体资源。该数据集通过 IDD 流水线将中文源端 45,518 个实体转换为 44,118 个维吾尔语实体，总体保留率为 96.92%，并在本研究自动审计范围内实现坏字符 span 数为 0。C2U-NER 的价值不仅在于规模，也在于它将维吾尔语 NER 中的后缀粘连、边界漂移、空样本误报和长实体漏检转化为可评测问题。基于该数据集，TBRK-IBP-CINO 在严格字符级评价下较 IBP-CINO 取得提升，并在三个公开来源整理得到的外部维吾尔语验证集上保持同向趋势。本文结果表明，低资源黏着语 NER 的后续进展需要同时推进可靠数据资源建设和边界感知模型设计。

## 附录A IDD 构建细节与审计规则摘要

为提高 C2U-NER 的可复现性，IDD 流水线在附录中被拆解为可记录的输入、输出和审计步骤。附录不依赖未公开的模型内部状态，而是说明外部研究者复现同类流程时需要固定的接口、字段和过滤规则。正式发布时，完整脚本、提示模板、运行参数和字段说明将与数据卡一并提供；若源端许可限制文本公开，仍可发布不含受限文本的模板、伪样例、哈希和统计摘要。

**A.1 零样本汉维翻译接口。** 每条源端样本包含 `sample_id`、`zh_text` 和 `zh_entities` 三个核心字段；其中 `zh_entities` 由 `(start, end, type, text)` 组成。Hunyuan-7B 翻译提示固定为“将中文新闻句子翻译为自然维吾尔语，保留人名、地名、机构名的核心语义，不额外解释，不输出列表或注释”。模型输出只接受单个维吾尔语句子字符串。译文进入后续阶段前需要去除多余引号、项目符号、解释性前缀和非文本控制符，并记录译文长度和是否为空。

**A.2 占用掩码硬对齐。** 硬对齐阶段对每个源端实体维护候选维吾尔语译名集合，并在目标句中搜索连续子串。若候选实体在目标句中出现多次，优先选择未被占用且与源端实体顺序最接近的 span；若多个候选具有相同优先级，则选择最长合法匹配。每个被接受的 span 必须满足四个条件：起点和终点位于目标句范围内；实体文本等于目标句切片；span 未与已接受实体发生非法重叠；实体类型属于 PER、LOC 或 ORG。硬对齐失败不会直接删除实体，而是进入软对齐队列。

**A.3 形态感知软对齐接口。** Qwen3-80B 软对齐阶段的输入为中文句子、维吾尔语句子、当前源端实体、实体类型、已占用 span 列表和硬对齐失败原因。模型输出采用 JSON 结构：`{"entity_id": "...", "target_text": "...", "start": int, "end": int, "reason": "..."}`。系统只将 `target_text`、`start` 和 `end` 用于审计，`reason` 仅用于人工诊断。若模型给出的 `start/end` 与 `target_text` 不一致，则以目标句中可唯一定位的 `target_text` 为准；若无法唯一定位或存在占用冲突，则拒绝该候选。

**A.4 生成式回填约束。** 对于仍有缺失实体的样本，生成式修复提示要求模型在不改变已接受实体 span 的前提下，重写或局部修正维吾尔语句子，使缺失实体以自然形式出现。系统不允许模型新增源端不存在的 PER、LOC 或 ORG 实体，也不允许改变已接受实体的类型。修复后样本重新执行全量 span 解析；若修复导致已接受实体消失、类型冲突或句子语义明显偏离源句，则回退到修复前版本，并将该实体标记为未保留。

**A.5 XLM-R 判别式自举。** 判别式自举仅用于残差补召回。训练正例来自前四阶段通过所有审计的实体 span，负例来自同句非实体片段、实体邻近偏移片段和空样本候选片段。推理时，模型为剩余候选输出类型分布和置信度；只有当最高类型属于 PER、LOC 或 ORG，置信度超过验证集阈值，并通过连续子串、占用冲突和标签空间审计时，候选才会并入最终数据。该阶段不覆盖、也不修改前四阶段的高置信实体。

**A.6 冲突与异常样本处理。** C2U-NER 不保留嵌套实体和非法重叠实体。若两个候选 span 完全相同但类型不同，优先保留源端标注中的类型，并将冲突写入日志；若两个 span 部分重叠，则优先保留构建阶段置信度更高、长度更接近源端实体译名的候选。对无法判定的冲突，样本进入人工复核池。空样本不因没有实体而丢弃，因为空样本 false positive 是本文边界评测的重要错误来源。

**A.7 实体保留率定义。** 本文的实体保留率以源端实体数为分母，以目标端成功获得合法字符 span 的源端实体数为分子。若一个源端实体在目标句中被自然拆分为多个不连续片段，则不计为保留；若一个实体经后缀粘连或词序重组后仍可定位为连续目标片段，则计为保留。重复实体按源端实体实例计数，而不是按唯一字符串计数。该定义解释了表1和表7中的 65.34%、94.87% 与 96.92%。
